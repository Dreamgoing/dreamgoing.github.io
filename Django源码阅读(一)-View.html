Django源码阅读-View

主要涉及django settings模块加载，middle，router，orm核心模块D

使用Django框架开发也有挺长时间了，Django其模块化的设计有很多优雅的实现和扩展功能的支持，这使得开发者能够专注于业务功能的实现。本着庖丁解牛的精神，本文简单的对Django，DRF的源码做一简单的分析。

概述

Django Rest Framework是Django的一个扩展, 简称为DRF，将Django变成一个REST(representational state transfer) API。

本文注意点:

  • 将Django和DRF结合起来阅读，源码篇幅较大，因此例如参数合法性校验，等等摘抄的源码可能有删减，最好对照Django和DRF的源码阅读。
  • 在以面向对象，模块化，面向大工程的实践中多以Class Base View为主，因此对于Function Base View介绍较少，不过二者在核心上面也类似。

Django基类

# django/view/generic/base.py
class View:
    # 支持的http方法，派生类没有实现对应的方法则抛出HTTP405
    http_method_names = ['get', 'post', 'put', 'patch', 'delete', 'head', 'options', 'trace']
    
    # classonlymethod 继承自classmethod重写了classmethod的__get__方法保护as_view方法仅支持类调用，不支持实例调用,即在decorator的基础上成为了descriptor，增加了一层限制。
    @classonlymethod
    def as_view(cls, **initkwargs):
        # request-response的入口函数
        
        # ... 进行参数校验和提示
        def view(request, *args, **kwargs):
            self = cls(**initkwargs)
            if hasattr(self, 'get') and not hasattr(self, 'head'):
                self.head = self.get
            self.request = request
            self.args = args
            self.kwargs = kwargs
            
            # 在urlpattern中使用 .as_view, 返回对应的函数
            return self.dispatch(request, *args, **kwargs)
        
        view.view_class = cls
        view.view_initkwargs = initkwargs
        
        # update_wrapper(wrapper,warpped, ...) functools中的函数，即将wrapped中的doc，__dict__, __name__, 等传递给wrapper, functools中的tools函数主要是生成了update_wrapper的partial偏函数。
        update_wrapper(view, cls, updated=())
        update_wrapper(view, cls.dispatch, assigned=())
        return view
    
    def dispatch(self, request, *args, **kwargs):
        # 根据请求体中的method，获取定义在类中定义函数
        if request.method.lower() in self.http_method_names:
            handler = getattr(self, request.method.lower(), self.http_method_not_allowed)
        else:
            handler = self.http_method_not_allowed
        return handler(request, *args, **kwargs)
