加密算法相关知识

基本概念

对称加密算法Symmetric-key algorithm

对称加密算法又称为共享密钥加密算法, 即在加密和解密的过程中使用的密钥是相同的. 这样的优点是计算速度快,常见的算法有 DES、3DES、AES、RC5、RC6. 缺点是双方通信需要共享并知晓公钥.

非对称加密 public-key cryptography

非对称加密算法又称为公开密钥加密, 服务端生成一对私钥和公钥, 私钥保存在服务端, 公钥自由发布供任何人使用. 客户端的明文通过公钥加密之后, 服务端通过私钥解密. 非对称加密在加密和解密的过程中使用不同的密钥. 常见的非对称加密即为RSA

数字签名 digital signature

数字签名即为服务端将要传递的报文进行哈希之后, 使用私钥private-key进行加密, 进行传递. 通常有RSA-SHA-256这样的形式.

数字证书 certificate authority

数字证书由第三方可信的机构颁发, 包含了公钥和网站的一些基本信息.

SHA-1

secret hash algorithm 1 类似于MD4 和 MD5的一种信息摘要算法.

MD5 最后生成的摘要信息为16个字节, SHA-1最后生成的摘要信息为20字节.

SHA-1 是一种哈希算法, 并不是一种加密算法. 也就是说, 他并不能通过解密还原到原来的文本. 也就是说他是一种一次新的密码算法. ‘one-way’ cryptographic function. 哈希算法就是将数据浓缩成一个固定的大小的信息串. 提供一种不可逆, 唯一的哈希值.

SHA-1 算法有如下几个重要的应用: 哈希表, 完整性校验, 挑战握手认证, 数字签名等.

HMAC

Hash Message Authentication Code 哈希消息校验码

 

应用

  • challenge handshake authentication挑战握手认证, 即避免了明文传递密码, 客户端向服务器传递哈希后的密码, 这样降低了密码被拦截的风险.
  • anti-tamper 防篡改, 将哈希后的信息与原来的信息拼接在一起, 接受者接收到这些信息之后, 可以再一次通过再次哈希re-hash原来的信息与提供的哈希值进行比较, 如果匹配则提供的信息没有改变. 否则提供的信息发生改变. 这样可以通常用来确保在传输的过程中有无数据丢失data-loss.
  • digital signature 数字签名, 通过私钥对文档的哈希值进行加密, 从而生成文档的数字签名. 然后其他人可以通过公钥来解密以及带有数字签名的文档, 这样再次获得了数字签名中哈希值, 然后在计算现在文档的哈希值, 进行比较, 看文档是否被篡改.
  • git代码管理系统, 广泛的使用SHA-1算法进行一致性验证

注意

要注意的是, 哈希方法是为了快速计算而设计的, 并不是专门了存储加密的密码, 哈希方法可以通过暴力破解.

SHA-1在很多系统中已经广泛使用, 对于可能受到攻击的情况, google已研究出了SHA-1的碰撞攻击, 更加推荐使用SHA-256, SHA-3

CA浏览器SSL/TSL 证书不再使用SHA-1签名的证书.
