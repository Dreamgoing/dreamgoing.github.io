Docker基础用法

记录docker使用中的一些基础用法

Docker的优点

  • 更加高效的利用资源: 由于不同虚拟化的方式不同, Docker比传统的虚拟机有更高的资源利用率.
  • 更快的启动时间, Docker容器应用, 直接运行于宿主内核, 无需启动完整的操作系统, 因此可以做到秒级毫秒级的启动.
  • 完整的运行环境, Docker镜像提供了除内核外完整的应用环境, 确保了测试环境, 开发环境, 生产环境的一致性.
  • 持续交付和部署, 开发者通过Dockerfile来进行镜像构建, 结合持续集成系统Continous Integration进行集成测试.

三个基本概念

镜像(image)

操作系统分为用户空间和内核空间, 对于linux而言, 内核启动后, 会挂载root文件系统为其用户空间提供支持. 比如: 官方ubuntu:16.04就包含了构建ubuntu16.04最小系统的root文件系统.

Docker镜像是一种特殊的文件系统, 包含了容器运行时所需要的程序, 库, 资源, 配置外, 还包含了一些为运行时准备的配置参数(匿名卷, 环境变量, 用户等), 镜像不包含任何动态数据, 其内容在构建之后也不会再被改变.

分层存储使得镜像的定制和复用更为容易

容器(container)

镜像(image)和容器(container)的关系, 就像面向对象程序设计中的类和实例, 镜像是静态的定义, 容器是运行时的实体.

常用命令

  • docker pull <image>: 拉取镜像
  • docker ps: 查看docker中进程
  • docker image ls: 查看已有的镜像
  • docker image rm <image>: 删除镜像
  • docker info: 查看docker的信息
  • docker run -it ubuntu:16.04: 以交互式的方式运行镜像
  • docker run -d -p <docker:port>:<expose:port> image: 以daemon运行docker image， -p参数为将docker内部的接口映射到宿主主机端口, 前者为宿主端口，后者为容器内端口
  • docker build -t <tag> .: . 是在指定context上下文目录, 即将该目录下的内容打包交给Docker引擎以构建镜像

使用Dockerfile进行定制镜像

  • FROM: 指定基础镜像, 对于Docker的分层存储, 基础镜像为一层
  • RUN: 用来执行命令行命令, 通常使用&&和\将多层RUN合并为一层
  • CMD: 用来指定默认容器主程序启动时的命令
  • ENTRYPOINT: 在运行时指定启动程序的参数:(ENTRYPOINT["curl", "-s", "http://ip.cn"])
  • COPY: 将构建上下文(context)中源文件复制到新一层的镜像内
  • ADD: 为更高级的复制文件
  • ENV: 配置环境变量, 通过$<env variable name>来使用.
  • VOLUME: 定义匿名卷, 由于开发环境代码一直在变动，而且多人通过 git 协作，于是代码都是放在外面，构建一个运行环境的 image，然后代码部分用 volume 映射进去，方便随时调整. VOLUME, 可以理解为目录映射, 改变宿主主机数据即可影响容器数据.
  • WORKDIR: 相当于cd, 用于改变各层的工作目录的位置
  • EXPOSE: 暴露容器中的端口

注意:

  • RUN命令:启动一个容器, 执行命令, 提交存储层文件变更. 增加一层.
