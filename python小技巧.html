python技巧

用来记录一些python使用过程中的技巧

技巧

  • 使用字典(dict)或者列表(list), 当需要赋值初始化值时,为了防止污染, 使用deepcopy

  • staticmethod :静态方法, 不知道类中的具体细节, 比如实例,类等, 无 implicit first argument (self)

  • classmethod :类方法, 知道类中的具体细节, 可以对类进行操控, 有 implicit first argument (cls)

  • method:实例方法, 与实例对象有关, 有implicit first argument (self)

  • python 字符编码问题: python3 unicode, 当抓取javascript 页面之后,可以使用json.load进行转换(leetcode spider中一个bug)

  • MCVE: Minimal, Complete and Verifiable Examplex

  • GIL: global interpreter look, CPython解释器使用,保证了在任意时刻,只有一个线程在解释器中执行, 是python

  • 可以通过继承自不同的库中的类,来实现自己的功能, 例如: 继承自requests.Session, threading.Thread, threading.local 等

  • python 类中的命名规范 _fun() 为认定为私有成员变量

  • getattr(object, name,[,default]): 获取object.name, 如果不存在该对象,则返回result

  • python __subclasses__ 方法用来获取基类中所有下一层子类, 主要基类和子类均属于同一个模块

  • python __init__.py 用来作初始化, 或者导出模块中所有的信息, from .error import *

  • pycharm: 可以设定一系列的搜索

  • python multiprocess 与 multithread对于文件I/O的效率不同, multprocess的效率高于multithread

  • python Mutiprocessing: pool

    • pool.apply: 函数调用在一个独立的进程中运行, pool.apply 将会被阻塞,直到函数调用完成
    • pool.apply_async: 该函数立刻返回, 而不是等待结果完成, 可以调用get方法取回函数调用的结果, get会阻塞该函数, 即pool.apply(func, args, kwargs) 等价于 pool.apply_async(func, args, kwargs).get()
    • pool.apply_async具有一个回调函数callback function, 如果提供了回调函数, 当func返回时调用该回调函数,而不调用get函数
    • pool.map(f, iterable), 该函数将iterable分成不同的块提供给不同的进程调用,提高了效率

  • python的mock测试中,patch decorator的顺序必须要和传入单元测试的参数顺序一致

  • python的mock测试中, 如果mock的对象在不同的module使用mock.patch装饰器会出问题

  • python的mock测试中, 如果使用了dict操作, 推荐使用get而不是[]

  • python的mock测试中, 如果使用了mock装饰器, 则该装饰器作用于该函数的全部生命周期

  • python werkzueg FileStorage, 是对于传入文件的一层很薄的wrapper, 是与flask相对应

  • python werkzeug封装了FileStorage, 坑点在于如果requests上传的文件如果比较大, 则会转成tempfile而不是StringIO, 因此使用方法时使用read()而不是get_value, read()完成之后将文件游标 file_obj.seek(0)

  • python werkzeug中FileStorage,read()读完之后stream游标发生变化, 再一次读会出现错误

  • Apache 同步多进程, Nginx 异步事件处理(多个连接对应一个进程), Nginx反向代理服务器, 7层负载均衡

  • application/octet-stream被定义成任意的二进制数据类型

  • 小端:低尾端, 大端:高尾端

  • chr() 函数从unicode integer返回一个字符

  • StringIO实现了一个类似对文件操作的接口, 可以对于String buffer进行读写, 类似与memory file

  • Python list: append向列表后面添加对象, extend扩展列表,添加另一个可迭代对象中, 每个对象

  • 锁是操作系统中为了保证操作原子性而引人的概念, Python中RLock是一种可重入锁(reentrant lock), RLock

  • python __enter__ __exit__魔法方法主要用来实现with 上下文相关常用与DBconnection 或者其他 mock

  • __getattr__执行顺序为优先找类属性, 没有找到则调用__getattr__来查找类中嵌套的属性

  • dict.items返回(key,value)的list, 经过了复制(copy), dict.iteritems返回可迭代的对象, 比dict.items更高效

  • python from __future__ import absolute_import意味着python引用将会最上层的包而不是引用相对路径的包

  • python string split(a,b) a为分割符, b为最大需要split的步数

  • Python subprocess Popen可以为在类unix系统中执行命令, 通常用subprocess来交互

  • Python @property与Python @property.setter和Python @property.getter用来配合使用

  • Python descirptor不能成为实例化的属性, 因为descriptor需要存在于类中而不是存在于实例中

  • Python not: 通常用于检测值是不是false, 如: empty_list, empty_string, None, false is none: 用于检测和判断值是不是None

  • 使用**kwargs传递参数时, 函数中保险起见需要同时加入 **kwargs防止出现错误

  • Python or 关键字 a or b 优先返回a. 当a不为None, 0, '', empty时候

  • Python 可以通过setup.py 或者其他来安装自定义的一些模块

  • pip freeze 可以查看python 安装的module的version

  • Monkey patching: 即动态的在运行时期替换原来代码中的一些属性的方法

  • global关键字用来在本地作用域中声明全局的变量

  • StopIteration: 是iterator.next()的结束的信号

  • Python zip(): 1. 将多个可迭代的序列转化成tuple的序列, 2. zip(*)将tuple的序列转化成多个序列

  • Python map: 对序列进行操作, 返回并收集结果

  • Python filter: 主要对序列进行过滤操作

  • Python threading.pool.apply: 调用之后该线程会执行并阻塞, 如果想要更好的性能使用threading.pool.async_apply.

  • Python threading.pool.async_apply: 调用之后该线程不会运行直到返回结果, 通过callback来返回结果

  • Python Lock:只能被一个线程请求一次, 不能被多次请求

  • Python RLock: 可重入锁

  • Python ast模块通常用来解析抽象语法树

  • Python ast.literal_eval(<str>): 用来解析字符串到

  • Python中的 try-except-else 可以使用异常来控制流程, else之后的语句在没有异常时执行, 并且在final之前执行. 可以简化异常情况.

  • Python multiprocessing.dummy为python的多线程. 即同一个进程下面有很多个线程

  • Python multiprocessing为python多进程库, 即重新创建Fork了新的进程
